## 📐 Padrões e Convenções do Java

O Java possui regras estritas de nomenclatura e estrutura que ajudam a manter a consistência do código:

* **Nomes de Classes:** Devem utilizar o padrão *PascalCase* (toda palavra começa com letra maiúscula). Exemplo: `MinhaPrimeiraClasse`. O nome do arquivo físico no computador deve ser exatamente igual ao nome da classe pública.
* **Nomes de Métodos e Variáveis:** Devem utilizar o padrão *camelCase* (a primeira palavra começa com letra minúscula e as seguintes com maiúscula). Exemplo: `calcularResultadoFinal()`.
* **Blocos de Código:** Em Java, delimitamos escopos (de classes, métodos ou estruturas condicionais) utilizando chaves `{ ... }`.
* **O modificador `static`:** Indica que um método ou atributo pertence à classe em si, e não a um objeto específico. É por isso que o método principal de entrada do Java (`main`) é sempre declarado como `public static void main`.
